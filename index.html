How to Fix “No Bootable Device Found” After Installing a New NVMe SSD

This guide walks through the real causes of the error, from bad SSD seating to UEFI/Legacy boot mismatch, and gives you the steps to recover fast.

Jump to Fix Guide Open Markdown Source

I spent three hours last Tuesday chasing a dead boot sequence on a custom PC that had just gained a shiny new NVMe SSD. The machine reported No Bootable Device Found, and the worst part was that the drive was physically installed and looked fine. That kind of failure is maddening because it is rarely one thing. It is usually a mix of firmware settings, boot order confusion, missing boot files, or an M.2 slot issue.

Why it happens

  • The motherboard does not detect the drive.
  • The BIOS boot order points to the wrong device.
  • The SSD has no working Windows bootloader.
  • UEFI and Legacy boot modes are mismatched.

What to check first

  • Confirm the SSD appears in BIOS.
  • Verify the M.2 slot and seating.
  • Set the new drive as the primary boot target.
  • Switch between UEFI and Legacy if needed.

Step-by-Step Troubleshooting

1. Reset the firmware and power state

Shut the PC down, unplug the PSU, hold the power button for 10 seconds, and then reconnect power. If the board has a clear CMOS option, use it. A stale firmware state can make the machine misread the new drive after installation.

2. Re-seat the NVMe SSD

Make sure the drive is fully seated in the correct M.2 slot. Tighten the retention screw carefully and verify the heatsink is not shifting the drive. A partially seated SSD can be invisible to the board until you reseat it.

3. Check BIOS storage detection

Enter the BIOS and look for the drive under storage or boot devices. If it is not listed, try another M.2 slot or update the BIOS. Some older boards are picky about Gen4 NVMe SSDs in specific lanes.

4. Match the boot mode

If Windows was installed in UEFI, the board should boot in UEFI mode. If it was installed in Legacy, the board must match that mode. Mixed settings are one of the most common causes of this exact boot error.

5. Set the correct boot order

Move the SSD to the top of the boot priority list. If you still have the old drive installed, disconnect it temporarily to force the machine to use the new one.

6. Repair Windows boot files

If the SSD is detected but still not bootable, repair the boot files with Windows Recovery. Use the installation USB and run bootrec /fixboot and bootrec /rebuildbcd if needed.

Symptom Likely Culprit
BIOS shows no drive at all Bad seating, wrong M.2 slot, unsupported SSD, or BIOS detection issue
Drive appears in BIOS but system still fails Wrong boot order, missing EFI partition, or broken bootloader
PC boots from old drive instead of new one Old drive still has higher boot priority
System throws the error right after install Firmware reset needed or M.2 slot disabled in BIOS

7. When a clean reinstall is the smarter move

If the clone was bad, the EFI partition is missing, or the machine is acting unstable, reinstalling Windows on the new SSD is often faster and cleaner than fighting a broken boot chain.
